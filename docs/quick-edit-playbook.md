# Quick Edit Playbook

Короткая инструкция для следующих чатов и мелких правок. Цель - не изучать Tilda export заново и не тратить несколько итераций на простые сдвиги.

## Почему правки тут могут быть медленными

Главная страница - это не обычная ручная верстка. Это Tilda Zero Block export внутри Astro. Большинство элементов стоят абсолютными координатами и имеют отдельные значения под брейкпоинты `1920`, `1200`, `640`, `320`.

Из-за этого простое "подвинь строку" технически означает:

1. найти правильный `rec...`;
2. найти правильный `data-elem-id`;
3. проверить desktop/tablet/mobile, потому что координаты разные;
4. убедиться, что Tilda runtime после загрузки не пересчитал блок иначе;
5. не задеть соседние элементы, которые тоже стоят абсолютными координатами.

Если править только один viewport или только один `top`, сайт может выглядеть нормально первые секунды или на одном экране, но поплыть на другом.

## Быстрый алгоритм любой визуальной правки

1. Прочитать `AGENTS.md`, этот файл и `docs/tilda-zero-editing.md`.
2. Найти текст/элемент через `rg` по `files/page62008353body.html` и `src/pages/index.astro`.
3. Зафиксировать `rec...` и `data-elem-id` элемента и соседей.
4. Править минимально в `src/pages/index.astro`, в CSS override или Tilda-like injected markup.
5. Если это контакты, popup, меню, телефон, Telegram/MAX, Instagram/VK или footer - обязательно запустить `npm run verify:contacts`.
6. Если это другая визуальная правка - минимум `npm run build`; если есть риск геометрии, добавить маленькую проверку в `scripts/check-contact-layout.mjs`.
7. После browser/headless/preview проверки закрыть тестовые процессы.
8. Только после успешной проверки: commit, push в `main`, deploy.

## Нельзя

- Нельзя двигать элементы runtime JS-ом после загрузки.
- Нельзя считать desktop достаточной проверкой.
- Нельзя добавлять обычный `position:absolute` поверх Zero Block без Tilda-like координат или CSS по брейкпоинтам.
- Нельзя коммитить временные PNG-скриншоты.
- Нельзя закрывать обычный Chrome пользователя без прямой просьбы.

## Что проверяет страховка контактов

`npm run verify:contacts` собирает сайт, запускает preview, открывает отдельный headless Chrome и проверяет `1911x1064`, `1440x900`, `390x844`.

Проверка ловит:

- Telegram/MAX отсутствуют, серые или уехали;
- строка контактов пересекает телефон или соцсети;
- MAX обрезан справа;
- стрелка и Instagram/VK не выровнены;
- Instagram/VK слишком далеко друг от друга;
- мобильный popup уводит телефон слишком низко;
- скрытые пункты popup снова видны;
- мобильная почта отсутствует или слишком далеко от соцсетей;
- мобильное footer menu растянуто или залезает на ИП/ОГРНИП/ИНН.

## Быстрый sanity-check после проверки

```bash
ps aux | egrep "contact-layout-chrome|remote-debugging-port=|headless=new|astro preview|npm run preview" | grep -v egrep
```

Если команда ничего не выводит - тестовых процессов нет.
