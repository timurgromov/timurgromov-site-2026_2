# Tilda Zero Editing Guide

Этот сайт выглядит как обычная страница, но главная страница технически остается Tilda Zero export, обернутый в Astro. Поэтому простая вставка HTML может "поплыть" после загрузки Tilda runtime.

## 1. Где что лежит

- `files/page62008353body.html` - исходный body export главной страницы.
- `src/pages/index.astro` - точка трансформации export: скрытие блоков, подмена видео, инъекция CSS/HTML.
- `public/css/` и `public/js/` - Tilda runtime и стили.
- `scripts/check-contact-layout.mjs` - проверка геометрии контактов в браузере.
- `scripts/verify-contact-layout.mjs` - безопасный запуск preview + проверка + cleanup.

## 2. Почему элементы "плавают"

Tilda Zero Block после загрузки делает `t396_initialScale` и `t396_init`. Эти функции пересчитывают элементы по `data-field-*` и брейкпоинтам. Если добавить обычный абсолютный `div` без Tilda-like атрибутов или двигать его JS-ом после загрузки, элемент может сначала выглядеть правильно, а потом уехать.

## 3. Правильный способ менять сетку

1. Найти record: `rec...`.
2. Найти соседний родной элемент: `data-elem-id`.
3. Скопировать координатную модель: `data-field-top-value`, `data-field-left-value`, `data-field-top-res-1200-value`, `data-field-left-res-1200-value`, ширина, font-size.
4. Добавлять новый элемент как `.t396__elem tn-elem` с уникальным `data-elem-id`.
5. Для CSS использовать те же брейкпоинты Tilda: `1920`, `1200`, `640`, `320`.
6. Не использовать runtime-пересчет координат от телефона, если можно задать фиксированные позиции по брейкпоинтам.

## 4. Контакты / Telegram / MAX

Текущие кастомные элементы:

- popup/menu shortcut: `#rec862707608 [data-elem-id="1746200000001"]`
- footer shortcut: `#rec862623921 [data-elem-id="1746200000002"]`

Рядом с ними важные родные элементы:

- popup phone: `1738924888767`
- popup social text: `1738924888797`
- footer phone: `1738908167577`
- footer social text: `1738908201655`

Правило: строка `Написать мне Telegram MAX` должна стоять ниже телефона и выше блока соцсетей. Цвет ссылок должен быть `#fffefa`.

## 5. Обязательная проверка

После любых изменений контактов, popup/menu или footer:

```bash
npm run verify:contacts
```

Эта команда:

1. собирает сайт;
2. запускает локальный Astro preview на временном порту;
3. открывает отдельный headless Chrome;
4. проверяет popup и footer на `1911x1064` и `1440x900`;
5. закрывает Chrome и preview.

Проверка падает, если:

- нет `Telegram` или `MAX`;
- ссылка не белая;
- строка пересекается с телефоном;
- строка слишком далеко от телефона;
- строка пересекается с соцсетями;
- `MAX` обрезан справа;
- строка выходит за оранжевый блок.

## 6. Cleanup Rule

Если вручную запускались preview/headless/browser automation, в конце работы выполнить:

```bash
ps aux | egrep "headless|remote-debugging-port|astro preview|npm run preview" | grep -v egrep
```

Если есть тестовые хвосты - остановить только их. Обычный Chrome пользователя не трогать без прямой просьбы.

## 7. Definition Of Done

Правка считается готовой только когда:

1. код изменен минимально;
2. `npm run build` прошел;
3. профильная проверка прошла;
4. preview/headless процессы закрыты;
5. commit и push в `main` выполнены;
6. опубликованная страница проверена или хотя бы HTML GitHub Pages содержит новый маркер.
